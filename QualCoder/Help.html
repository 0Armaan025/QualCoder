Overview

QualCoder is used for qualitative data analysis. All project files are stored in a project.qda folder.

Text files can be imported from plain text, docx, odt and pdf format. Image files from jpg, png and gif can be imported. Text files can also be entered manually. Survey spreadsheets in csv format can be imported.

Text can be assigned codes. Codes can be ordered into categories. Categories can be stored within a hierarchical tree structure, so several similar categories can be assigned into overarching categories. Annotations can be added to the text. Various code reports can be generated. Reports can be exported as plain text or html files. Codes and categories can be renamed, merged and deleted. Memos can be assigned to the entire project, codes, files, cases and attributes.

Project menu

Create new project

Create a new project. The project is stored in a projectname.qda folder. The project folder contains sub folders including folders for documents, images and (for the future) audio files.

Open project

Use this option to select and open a project. When opening a project, select the projectname.qda folder and press the open button.

Close project

Use this option to close a currently open project.

Project memo

General notes about the current project can be entered and viewed here.

Settings

There are several options in the settings screen. The current coder name can be entered here, or it can be left as default. A second coder name can be entered. Choice of coder for the current session can be changed via the drop down list. Settings and Log files are located in the user's home directory.

Font and font size can be changed. It is recommended that the font size for the code trees be fairly small e.g. size 10. A default directory can be entered in the settings also.

Files and cases menu

Manage files

The Manage files dialog shows any files that have been loaded. New files can be loaded using the Import button. Files can be imported from plain text, .docx, .odt and pdf. Some pdf files may not import very well. Image files can be imported from jpg, jpeg, png and gif. These are stored in the qda/images folder.

Memos can be added to each file. The memo square says Yes indicating a memo exists. To delete any memo, open the memo, delete all text from it and click the OK button.

File names can be changed by double-clicking on the file name and entering a new name.

New text files can be created by pressing the Create button.

File attributes can be added to describe and classify the files.

Text files can be exported to plain text and files can be deleted.

Manage cases

Cases are useful for assigning attributes such as age and sex. You can add a memo to each case and rename a case by double-clicking on a case name. Cases can be manually added.

Cases can be imported (as a once off procedure) from a csv file that contains all the cases and any attributes. The first column of the csv file must have the case ids and the file must have a header row. Each of the remaining columns (if any) contain the attribute name and attribute details for each case.

The View case button shows all the text in all the files that is associated with the selected case. It also identifies (but currently does not display) the image files.

You can assign all the text of a file to a case by selecting a case then clicking Add file to case. Image files can be added to a case.

The Open file button show the text of a file. Any currently selected case text will be underlined in red.

Adding and removing text assigned to a case:

Open a file to view it. The file will show in the right hand screen. Select a case then select text with the mouse. Right click to Mark this text to assign it to the case. Similarly, with marked text, you can Unmark using the mouse right click.

You can automatically assign portion of file text to a case by selecting a case then clicking the Auto assign file text button. You will be asked which file or files to assign the case to. Next you need to enter the start and end marks. An example may be a start mark of '[id1]' and an example end mark may be '['.

Cases can be deleted using the Delete case button.

Manage Journals

Select Manage journals. You can create journals to write down your thoughts and ideas when investigating the text and developing themes. Journal entries are dated and can be exported or deleted.

Manage Attributes

Attributes provide additional detail about files or cases. The Manage Attributes dialog allows manual addition or deletion of attributes. Each attribute can be assigned to either the cases or the files. Attributes can be of a character or numeric type. Memos can be given for attributes. Double-click on an attribute name to change the attribute name.

Import survey

Surveys must be in csv file format. A dialog box will open and the survey delimiter must be chosen. The delimiter is typically a comma ‘,’ but can be a tab or other character. Type ta, tb or tab to tell QualCoder the file is in tab format. Select a quote format: NONE means no quote, MINIMAL means text is quoted but numbers are not, and ALL means every variable of interest is quoted.

Coding menu

Code text

This is the central dialog for assigning codes to text. Once text segments are coded, hovering the mouse over the coded text shows the code name as a tooltip. Clicking on the coded segment also shows the code name.

The left hand side shows existing codes. Codes can be assigned a colour and moved under other codes in a tree-like hierarchy. Codes can be assigned memos.

View File

To begin coding you need to view some text. Select a file to view from the list presented in this dialog.

Code tree

Right click to add codes. Memos can be added and are used for the basis of a codebook. Codes can also be assigned a colour. If no colour has been selected the text is marked with a gray colour.

To delete a code, select a code, right click, and choose delete. Right click to rename a code or change a code colour. Similarly, right click to add, rename, delete or merge code categories.

You can drag and drop codes into different categories. Dragging a code onto another code will merge the two codes together.

Auto Code

Select a code then click the Auto Code button. Enter the text you want to autocode. The next dialog asks for one or more files to autocode. All matching text will be assigned the selected code.

Mark and unmark text

Select a file then select code. Highlight some text, right click and choose mark to assign the code to the text.

Add an annotation (like a memo) to the text. Select some text, right click and choose annotate from the menu. All the annotated text will become bold to mark the position of the annotation. To remove an annotation, open the annotation and delete all the text in your notes.

To remove a code from text, click on the coded text then right click and choose unmark.

Search text

Enter text in to the text box then click on the button to jump to each spot in the text that matches your selection.

Code image

Select an image. Once selected, the image will be shown in the screen. Any coded image areas will be marked with a red box. Hovering the mouse over a coded area will show the code in a tooltip.

An image can be scaled down to a smaller size using the slider at the bottom of the dialog.

To mark an image area, select a code, then with the left mouse button, mark an area of the image. You can add a memo by right clicking and choosing Memo.

You can remove a coded area by right clicking inside the coded area and selecting Unmark.

Code tree

Right click to add codes. Memos can be added and are used for the basis of a codebook. Codes can also be assigned a colour. If no colour has been selected the text is marked with a default colour.

To delete a code, select a code, right click, and choose delete.

Right click to rename a code or change a code colour.

Similarly, right click to add, rename, delete or merge code categories.

Codes and categories can be dragged to merge or be assigned under other categories.

View graph

A node graph of categories and codes is displayed. Various options are available including selection of particular categories. Nodes can be moved around using the mouse for easier viewing. Right-clicking on a node will display the relevant coded text or memo. Node text can be edited for easier viewing, but clicking on the node and typing any changes. The graph cannot be exported, but you can take a screenshot.

Export Codebook

A codebook is a list of your codes. Each code memo should detail the reasons behind assigning each code to the text. The codebook is exported to a text file.

Reports

Coding reports

This report gives a list of coded text. One or more codes or categories need to be selected and one or both of coders need to be selected. The Search button will present the results.

If a category is selected all codes in that category are also selected. Multiple categories can be selected using the mouse and the 'ctrl' button. Codes and categories can be sorted to help find the codes of interest.

Codings can be narrowed down by using the Search Text function or the File selection button. Only codings within the selected files will be presented, or if this function was not used all files are used. If text is entered into the Search text field, only codings which contain the matching text will be presented.

Codings can also be narrowed down to case selections using the Case selection button. Note that when case selections are used, file selections are ignored. Also when file selections are used, case selections are ignored.

Codings can also be narrowed down using case and file attributes. For example, selecting participant interview files and selecting only those participants who were over 50 years old.

Reports can be exported to text, open document format (ODT) or html files.

SQL statements

This dialog contains three panes. The top pane is where SQL statements are entered and the bottom pane contains the results of queries. The left pane contains tables and field names. Double-clicking on a field name adds it to the SQL statement. Results can be exported to a text file. If you are not familiar with SQL take care as you will be able to update and delete the data as well as select data.

Most fields are text. Fields ending in id, x1, y1, width, height, pos0 and pos1 are integer.

Selection results can be exported as a delimited file.

Coding comparison

This dialog allows you to compare the coding of two coders for text codings. The comparisons can be for all codes or for selected codes and categories. Percentage agreement and Cohen’s Kappa are calculated. Results can be exported as a text file.
